CFileCaptureBuffer Class

The CFileCaptureBuffer class is used for buffering packets that will be saved periodically to disk. CFileCaptureBuffer multiply inherits from two classes: CCaptureBuffer and CRefreshFile. The buffering capability for CFileCaptureBuffer comes from CCaptureBuffer and the file saving criteria are established via methods inherited from CRefreshFile.

Public Member Functions Inherited From CCaptureBuffer

A CFileCaptureBuffer object has available all of the CCaptureBuffer member functions due to the inheritance hierarchy. Note, however, that the following functions defined in CCaptureBuffer are overridden by CFileCaptureBuffer and the documentation for them should be taken from the present CFileCaptureBuffer document: Copy, Reset, Save, SaveSelected, and WritePacket.

Otherwise all the public member functions for CCaptureBuffer can be referenced from that document and are available for use by CFileCaptureBuffer objects.

Public Member Functions Inherited From CRefreshFile

A CFileCaptureBuffer object has available all of the CRefreshFile member functions due to the inheritance hierarchy. Note, however, that the following functions defined in CRefreshFile are overridden by CFileCaptureBuffer and the documentation for them should be taken from the present CFileCaptureBuffer document: Copy, Reset, and SetRefreshSize.

Otherwise all the public member functions for CRefreshFile can be referenced from that document and are available for use by CFileCaptureBuffer objects.

CFileCaptureBuffer::CFileCaptureBuffer

Constructs a CFileCaptureBuffer object based on a set of input arguments.

CFileCaptureBuffer(
  CString strFilePath,
  CString strFileBaseName = _T(""),
  UInt32 nRefreshFileType = kRefreshFileTypeNone,
  bool bAppendTimestring = false
);

Parameters

strFilePath
The file path that the packet file will be saved to.
strFileBaseName
An optional base name to build a file name from (could be appended to with a formatted timestamp, for example).
nRefreshFileType
An optional parameter specifying refresh file type. Possible values include: kRefreshFileTypeNone, kRefreshFileTypeOverwriteExisting, kRefreshFileTypeFileNumber, kRefreshFileTypeTimeStampWithNumber, and kRefreshFileTypeTimeStamp.
bAppendTimestring
True causes a formatted time string to be appended to the file name.

Remarks

This constructs a CFileCaptureBuffer object. The parameters specify file path, a base file name, the type of refresh, and whether or not to append a formatted time string value to the file base name. As an example if strFilePath = "C:\MyPath", strFileBaseName = "TestFile", nRefreshFileType = kRefreshFileTypeTimeStamp, and bAppendTimestring = true, then the resulting file name might be "C\MyPath\TestFile2010-04-12T15.42.48.752Z.pkt".

CFileCaptureBuffer::Monitor

Performs a Refresh if time has expired.

virtual bool Monitor(
  bool bWriteIfEmpty = true
);

Parameter

bWriteIfEmpty
True (the default value) indicates that the file write should be performed even if the buffer is empty.

Return Value

Returns true unless a file refresh fails.

Remarks

This method performs a Refresh, i.e. a write to disk of the buffer contents, if the time expiration value has been exceeded. If the time has not expired, the function simply returns true.

CFileCaptureBuffer::Refresh

Writes all buffer contents to disk, then clears the buffer contents.

virtual bool Refresh(
  bool bWriteIfEmpty = false
);

Parameter

bWriteIfEmpty
True indicates that the file write should be performed even if the buffer is empty. The default value is false.

Return Value

Returns true unless the file write fails.

Remarks

This method performs a Save to disk of the contents of the buffer. Once the buffer contents (i.e. all packets) have been written to the disk file then a Reset of the buffer is performed.

CFileCaptureBuffer::Reset

Clears all contents of the buffer and resets (zeroes) associated data.

void Reset();

Remarks

This method clears the contents of the buffer and zeroes out all data in the CFileCaptureBuffer object.

CFileCaptureBuffer::Save

Creates a new file name and writes the buffer contents to the new file.

bool Save(
  bool bWriteIfEmpty = false
);

Parameter

bWriteIfEmpty
True indicates that the file write should be performed even if the buffer is empty. The default value is false.

Return Value

True if the write to disk succeeds. Also returns true if the buffer is empty and the bWriteIfEmpty parameter is false.

Remarks

This method first creates a new file name based on the parameters that have been set controlling file name creation. Then the contents of the buffer are written to the new file. Note that this function in itself does not clear the buffer. The function to call for performing both a save and a clearing of the buffer is Refresh.

CFileCaptureBuffer::SaveSelected

Saves a specifed range of packets to disk file.

bool SaveSelected(
  UInt32 inRangeCount
  UInt64* inRanges
);

Parameters

inRangeCount
The number of range values that are included in the inRanges array.
inRanges
An array of range pairs, each pair being the first and last packet number to be selected.

Return Value

Returns true unless the buffer is empty or the destination path does not exist.

Remarks

This method saves "selected" packets to disk file. The file to be saved to is created new based on the file naming parameters that have been specified for the CFileCaptureBuffer object. The parameters specify the ranges of consequtive packets to be saved. For example, if three ranges of packet numbers are to be saved and the ranges are to consist of packets 2-4, 50-100, and 200-220, then the inRangeCount value would be 6 (specifying that there are 6 range values), and the inRanges array would be: {2,4,50,100,200,210}.

CFileCaptureBuffer::SetRefreshSize

Sets the packet buffer size limit that triggers a Refresh.

bool SetRefreshSize(
  const UInt32 inRefreshSize
  const UInt32 inUnits
);

Parameters

inRefreshSize
The size to be established for controlling refreshes, based on the inUnits parameter.
inUnits
Unit measure for setting the refresh size. This can be kKilobytes, kMegabytes, or kGigabytes as defined in RefreshFile.h

Return Value

Returns true unless the buffer is empty or the destination path does not exist.

Remarks

This method sets the packet buffer size limit that triggers a Refresh. The refresh size is based on the inRefreshSize parameter as the number of units, and the inUnits parameter as the type of units. For example, if inRefreshSize is 5 and inUnits is kMegabytes, then the refresh size in bytes will be calculated as (5) * (1 megabyte) = 5 * 1048576 = 5242880 bytes. This method internally calls CCaptureBuffer::SetCapacity to set the actaul buffer size (capacity) according to the specified refresh size.

CFileCaptureBuffer::WritePacket

Adds a packet to the buffer. The packet's header and packet data are written into the buffer.

void WritePacket(
  CPacket& inPacket
);

Parameter

inPacket
Reference to a CPacket to write into the buffer.

Remarks

This method writes a CPacket into the file capture buffer. This is a packet copy in that that CPacket's data as well as header are written into the buffer, not just referenced. Meta data is also maintained for the packet. Note that before writing the packet a check on the refresh specifications is made. If specified value for time elapsed or size limit is exceeded then a Refresh is performed before adding the new packet.
