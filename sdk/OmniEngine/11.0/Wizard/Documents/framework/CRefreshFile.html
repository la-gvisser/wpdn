CRefreshFile Class

The CRefreshFile class offers the methods concerned with refreshing a file. This class does not function in a standalone context (it has some virtual functions), but is used in the inheritance hierarchy for such classes as CFileCaptureBuffer to give it robust packet file naming and managing capability.

CRefreshFile::CRefreshFile

Constructs a CRefreshFile object (default constructor).

CRefreshFile();

Remarks

This constructs a CRefreshFile object. Before meaningful use, methods should be called to set the file path, file base name, and other attributes as well.

CRefreshFile::CRefreshFile

Constructs a CRefreshFile object based on a set of input arguments.

CRefreshFile(
  CString strFilePath,
  CString strFileBaseName = _T(""),
  UInt32 nRefreshFileType = kRefreshFileTypeNone,
  bool bAppendTimestring = false
);

Parameters

strFilePath
The file path that the file will be saved into.
strFileBaseName
An optional base name to build a file name from (could be appended to with a formatted timestamp, for example).
nRefreshFileType
An optional parameter specifying refresh file name type. Possible values are enumerated here.
bAppendTimestring
True causes a formatted time string to be appended to the file name.

Remarks

This constructs a CRefreshFile object. The parameters specify file path, a base file name, the type of refresh, and whether or not to append a formatted time string value to the file base name. As an example if strFilePath = "C:\MyPath", strFileBaseName = "TestFile", nRefreshFileType = kRefreshFileTypeTimeStamp, and bAppendTimestring = true, then the resulting file name might be "C\MyPath\TestFile2010-04-12T15.42.48.752Z.pkt".

CRefreshFile::Activate

Triggers a class that derives from CRefreshFile to begin buffering packets and periodically flushing them to a disk file.

bool Activate();

Return Value

Returns true unless the refresh file type has been set to kRefreshFileTypeNone, in which case returns false.

Remarks

This method activates (triggers) the buffering and refreshing process for a class such as CFileCaptureBuffer that derives from CRefreshFile. The derived class will not buffer packets or save them to disk file until the Activate method has been called.

CRefreshFile::Copy

Copies the CRefreshFile internal data from one CRefreshFile-derived object to another.

bool Copy(
  const CRefreshFile& inRefreshFile
);

Parameter

inRefreshFile
Reference to the CRefreshFile-derived object being copied from.

Return Value

Always returns true.

Remarks

This method copies the internal CRefreshFile data from object to another. Since the CRefreshFile class is always used as a base class for another class such as the CFileCaptureBuffer class, the copy operation for CRefreshFile will be tied to the copy operation for the derived class.

);

CRefreshFile::SetRefreshTimeInterval

Sets the time value for establishing when the next time-based refresh will occur.

bool SetRefreshTimeInterval(
  UInt32 inTimeInterval,
  UInt32 inTimeUnits
);

Parameters

inTimeInterval
The number of time units (seconds, minutes, etc.) to used to signal the next refresh.
inTimeUnits
The type of units to be converted.

Return Value

True unless time values are out of bounds (see Remarks).

Remarks

This method sets the time interval that controls time-based file refreshes. Once time elapsed since the last refresh exceeds this value a refresh is triggered. Note that refreshes may be triggered by time elapsed, file size exceeded, or by either (whichever comes first). This means that setting a time expiration value is optional. The minima and maxima for the refresh time interval are: minimum of 30 seconds and maximum of 30 days.

This method sets the time interval, expressed in milliseconds, that controls time-based file refreshes. Once time elapsed since the last refresh exceeds this value a refresh is triggered. The minima and maxima for the refresh time interval are: minimum of 30 seconds and maximum of 30 days (both expressed as milliseconds).

CRefreshFile::GetRandomizedMilliSec

Returns a randomized value that can be used as the milliseconds part of a formatted time string.

static WORD GetRandomizedMilliSec(
  bool inSeed = false
) throw( ... );

Parameter

inSeed
True indicates that the randomizing algorithm use an extra initial seed value based on the GetTickCount function.

Return Value

A randomized WORD value between 0 and 999.

Remarks

This static "helper" method creates a randomized value between 0 and 999 that is considered "milliseconds". This means that the generated value can be used as the milliseconds portion of a formatted time string. This in turn will help guarantee that two files whose names are based on the current time (via the GetTickCount function), and which are triggered to flush to disk simultaneously, will still have differing names.

Supplemental Information

Refresh File Name Type Indicators

The following types identify which algorithm will be used to create a new file name.

enum {
  kRefreshFileTypeNone = 0,
  kRefreshFileTypeOverwriteExisting,
  kRefreshFileTypeFileNumber,
  kRefreshFileTypeTimeStampWithNumber,
  kRefreshFileTypeTimeStamp,
  kRefreshFileTypeDefault = kRefreshFileTypeTimeStamp
};
kRefreshFileTypeNone
No file name will be created.
kRefreshFileTypeOverwriteExisting
Uses only the file path and the base file name. Thus the new file, having an identical name to the prior one used, will overwrite any existing file that was already created with this algorithm.
kRefreshFileTypeFileNumber
Creates a file name based on file path and base file name plus an integer value that is incremented by one each time a new name is created.
kRefreshFileTypeTimeStamp
Creates a file name based on file path and base file name plus an unique formatted timestamp value.
kRefreshFileTypeTimeStampWithNumber
Creates a file name based on file path and base file name, plus an unique formatted timestamp value, plus an additional integer value that is incremented by one for each new name created.

Refresh File Time Units

Following are the time unit types for conversion to milliseconds:

enum {
  kTimeUnitMilliseconds = 0,

  kTimeUnitHours,
